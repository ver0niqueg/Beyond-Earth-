Guide Pas à Pas — Beyond Earth
De zéro à un projet portfolio complet · ~3 semaines
ESTIMATED TIME
15–20h
01
/ 06
COLLECTE · 1-2h
Télécharger les données NASA
Rends-toi sur exoplanetarchive.ipac.caltech.edu → "Confirmed Planets" → Download CSV. Tu obtiens un fichier avec +50 colonnes : masse, rayon, période orbitale, distance, méthode de détection, année de découverte, température...
Créer un dossier projet Sauvegarder le CSV brut Lire la documentation des colonnes
Navigateur Excel (première vue)
02
/ 06
NETTOYAGE · 3-4h
Explorer & nettoyer avec pandas
C'est l'étape la plus importante — et celle que les recruteurs regardent. Le dataset a beaucoup de valeurs manquantes (NaN). Tu vas apprendre à les identifier, décider quoi en faire, et documenter tes choix dans un Jupyter Notebook.
df.info() · df.describe() Visualiser les NaN avec missingno Filtrer les colonnes utiles Renommer les colonnes Typage des dates
Python · pandas Jupyter Notebook
03
/ 06
ANALYSE SQL · 2-3h
Requêter les données avec SQLite
Importe ton DataFrame nettoyé dans une base SQLite. C'est simple avec pandas (df.to_sql()). Ensuite, écris des requêtes pour répondre à tes questions analytiques.
Top 10 planètes par taille COUNT par méthode de détection AVG distance par année Filtrer la zone habitable
SQLite pandas to_sql() DBeaver (optionnel)
04
/ 06
VISUALISATION · 4-5h
Créer 5 visualisations clés
Chaque graphique doit répondre à une question précise. Évite les graphiques "jolis mais vides". Pour chacun, écris 2-3 lignes d'interprétation dans ton notebook.
Bar chart — méthodes de détection Line chart — découvertes/an Scatter — masse vs période orbitale Histogram — distribution des rayons Carte — zone habitable
matplotlib seaborn plotly (interactif)
05
/ 06
FRONT-END · 3-4h
Créer un dashboard web interactif
C'est ici que tu montres tes compétences front-end ! Crée une page HTML/CSS/JS qui présente tes résultats de façon visuelle et professionnelle. Tu peux utiliser Chart.js pour les graphiques interactifs ou même React si tu te sens prêt.
Design thème spatial (dark mode) KPI cards en haut Graphiques interactifs Chart.js Responsive (mobile) Animations CSS au scroll
HTML / CSS / JS Chart.js React (optionnel)
06
/ 06
PUBLICATION · 1h
Publier sur GitHub + GitHub Pages
Mets tout en ligne pour que les recruteurs puissent le voir. Un bon README est aussi important que le code — explique le projet, les données, les conclusions, et les outils utilisés.
Créer un repo GitHub Rédiger un README soigné Déployer le dashboard sur GitHub Pages Ajouter le lien sur LinkedIn
